# 事业部业务管理系统 - 技术开发方案

## 一、技术架构推荐

### 推荐技术栈

| 层级 | 技术选型 | 理由 |
|:---|:---|:---|
| **前端** | Next.js + React | 成熟稳定，AI擅长，前后端一体 |
| **后端** | Next.js API Routes | 无需独立后端，简单 |
| **数据库** | Supabase (PostgreSQL) | 免费强大，无需运维 |
| **部署** | Vercel | 零配置，CDN加速 |
| **认证** | Supabase Auth | 内置用户/权限管理 |
| **版本管理** | GitHub | 用户已有账号 |

### 为什么推荐这套架构？

```
✅ 无开发经验也能上手
✅ 本地测试 = 云端部署（同一套代码）
✅ 免费额度够用（个人项目）
✅ AI工具最擅长生成此技术栈
✅ 安全合规支持（等保基础）
```

## 四、安全加固方案

### 4.1 等保测评基础要求

| 安全项 | 实现方式 |
|:---|:---|
| 身份鉴别 | Supabase Auth（强密码+验证码） |
| 访问控制 | RBAC角色权限（7个角色） |
| 安全审计 | 日志记录所有操作 |
| 数据传输 | HTTPS强制加密 |
| 数据存储 | 敏感数据加密存储 |

### 4.2 代码安全规范

```typescript
// ❌ 禁止：直接拼接SQL
const query = `SELECT * FROM users WHERE id = ${userId}`;

// ✅ 必须：使用参数化查询
const { data } = await supabase
  .from('users')
  .select('*')
  .eq('id', userId);

// ❌ 禁止：前端暴露敏感信息
const secretKey = 'sk-xxx'; // 错误！

// ✅ 必须：环境变量 + 后端API
// .env.local: SUPABASE_SERVICE_KEY=xxx
// 只在后端使用
```

### 4.3 安全检查清单

- [ ] 输入验证（所有用户输入）
- [ ] XSS防护（React默认防护）
- [ ] CSRF防护（Supabase内置）
- [ ] SQL注入防护（参数化查询）
- [ ] 敏感信息不提交Git
- [ ] 定期更新依赖包

## 七、版本管理方案

### 7.1 Git工作流

```
main分支 ──────────────────────────────→ 生产环境
    ↑
development分支 ──────────────────────→ 测试环境
    ↑
功能分支 (feature/xxx) ──────────────→ 开发
    ↑
修复分支 (fix/xxx) ──────────────────→ Bug修复
```

### 7.2 提交规范

```
feat: 新功能
fix: Bug修复
docs: 文档更新
test: 测试用例
refactor: 代码重构
chore: 构建/工具
```

### 7.3 GitHub Actions自动部署

```yaml
# .github/workflows/deploy.yml
name: Deploy
on:
  push:
    branches: [main]
jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - run: npm install
      - run: npm run build
      - uses: amondnet/vercel-action@v20
        with:
          vercel-token: ${{ secrets.VERCEL_TOKEN }}
          vercel-org-id: ${{ secrets.ORG_ID }}
          vercel-project-id: ${{ secrets.PROJECT_ID }}
```

## 八、部署方案

### 8.1 本地开发

```
1. 克隆代码
2. npm install
3. 配置.env.local
4. npm run dev
5. 浏览器访问 http://localhost:3000
```

### 8.2 云服务器部署

```
方案A：Vercel（推荐）
├── 优点：零配置、免费、全球CDN
├── 步骤：GitHub推送 → 自动部署
└── 成本：免费

方案B：阿里云/腾讯云
├── 优点：完全控制
├── 步骤：购买服务器 → 部署Docker
└── 成本：约100元/月
```
